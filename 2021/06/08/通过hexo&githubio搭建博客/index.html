1. Hexo概述

1.1 简介

Hexo 是一个博客框架， Hexo 使用 Markdown (或其他渲染引擎）解析文章，并生成静态资源站点。

1.3 项目的结构

初始化后的项目目录结构如下（linux上通过tree -L 1查看目录结构）

当前目录下需要关注的文件主要有：

  • _config.yml: 网站的主配置文件，配置主题，语言等;
  • scaffolds: 模板文件夹, 模板是指在新建的文章文件中默认填充的内容;
  • source: 资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去;
  • theme: 主题文件夹，用于存放Hexo的主题资源;

1.4 布局

Hexo 有三种默认布局：post、page 和 draft。在创建者三种不同类型的文件时，它们将会被保存到source中的不同的路径中，如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替，默认是scaffolds/post.md。

布局名称 存储位置 作用
post source/_posts 新建的文件默认采用该种方式布局
page source 生成新的页面，如about me页面
draft source/_drafts 草稿文件默认采用该种方式布局

3 GitHub Pages

GitHub Pages是一个静态网站托管服务工具。

3.1 创建GitHub Pages

介绍和操作可以查看GitHub Pages | Websites for you and your projects, hosted directly from your GitHub repository. Just edit, push, and your changes are live.

3.2 部署静态站点

将静态网站推送到刚才创建的站点，就可以通过username.github.io访问了， 推送静态文件时需要修改配置文件中的deploy相关内容（在站点的配置项中）。

关联后执行部署命令

部署时如果提示错误：ERROR Deployer not found: git， 则需要安装hexo-deployer-git
npm install hexo-deployer-git –save

访问GitHub Pages

如果无法访问GitHub相关网站，可以修改件C:\Windows\System32\drivers\etc\hosts或/etc/hosts文件，进行手动解析。

3.3 绑定自己的域名

目前访问的地址是github.io的子域名, 可以绑定自己的域名。相关操作可查看新手引导：教你如何解析域名？ - 知乎 (zhihu.com)。

4 图床使用

在使用hexo时，在生成的静态资源中引用一些图片不是很方便，可以使用图床^1进行图片的引用。在使用图床时可以使用PicGo [^2]工具管理，具体使用方法可以参考听说你也想用PicGo | PicGo
